            <section class="copy instapaper_body" itemprop="articleBody">
                                <p>
                <span class="embedded-Media-image layout-object img-caption-c">
	
			<img src="http://readwrite.com/files/styles/630_0su/public/fields/GithubTutorial_part2.jpg" style="" alt="How To Use GitHub Pages To Make Web Sites While Learning Code" width="1440" height="810">
	
	
	</span>
                </p>
                                <p class="p1">People use GitHub to host projects ranging from music composition to cookbooks to municipal data. But what you might not know is that GitHub isn’t just a storage locker, but also a showcase. That is, GitHub allows you to set up a bare-bones website you can use to display projects and collaborate.</p>
<p class="p1">Welcome to <a href="http://pages.github.com/">GitHub Pages</a>, the glossy, forward facing side of the site. Unlike the dense repositories that would confuse your less tech-savvy friends, Pages are ordinary looking websites that GitHub hosts for free.</p>
<p class="p1">GitHub Pages, which are&nbsp;a bit like Git with training wheels, let you explore repository building without ever leaving the website. No command line coding here: GitHub Pages lets you make a documented change, or "commit," to the site simply by clicking “OK.”</p>
<div class="layout-object right small">
<blockquote class="p3"><strong>See also: <a href="http://readwrite.com/2013/11/08/seven-ways-to-use-github-that-arent-coding#awesm=~ookowF8ZYmN0vz">Seven Ways To Use GitHub That Aren’t Coding</a></strong></blockquote>
</div>
<p class="p1">GitHub's&nbsp;<a href="http://matthewjmccullough.com/">Matthew McCullough</a>&nbsp;spends his time traveling the world teaching students to use Git and GitHub. He said that people who want to explore basic coding and collaborative concepts without having to get in too deep are&nbsp;ideal candidates for GitHub Pages.</p>
<p class="p1">“[Some people] want to click a button, get a themed template on WordPress, and they’re fine,” he said. “But I’ve noticed more and more people want to get their feet wet and understand what code is, and Pages is their gateway.”</p>
<h2 class="p4"><strong>Why Use Pages?</strong></h2>
<p class="p1">If you’re looking for an out-of-the-box website, Pages isn’t that, though McCullough hopes that the service will eventually have more user-friendly options.&nbsp;</p>
<p class="p1">Right now, people choose to use Pages for the same reason they choose to use GitHub—it’s ideal for collaboration. All Pages blogs are open source, so anyone can view the repository side of the site, look for bugs, and submit a pull request to the user. Of course, nobody can alter the site except the owner, who can choose to approve these pull requests or not.</p>
<p class="p1">As an example, McCullough shows off the GitHub Teaching site. <a href="http://teach.github.com/">From the Pages side</a>, it’s spartan, but familiar. But <a href="https://github.com/github/teach.github.com/blob/gh-pages/index.md">from the back side</a>, it’s a repository where students can learn by making edits to the very site that’s helping them learn.</p>
<p class="p1">“Culturally, why would people choose this?” McCullough posits. “It’s nerdy and sticky and less polished. But I would say the ability for people to find a public website and make contributions that then get recorded with their name on it makes it worth it. Open source, open culture.”</p>
<div class="layout-object left medium">
<blockquote class="p1"><strong style="line-height: 1.538em;">See also:&nbsp;<a href="http://readwrite.com/2013/09/30/understanding-github-a-journey-for-beginners-part-1">GitHub For Beginners: Don’t Get Scared, Get Started</a></strong></blockquote>
</div>
<p class="p1"><span style="line-height: 1.538em;">And of course, the fact that anything you put on GitHub remains always and entirely your own property and copyright doesn’t hurt. (See Section F of the</span><span style="line-height: 1.538em;">&nbsp;</span><a style="line-height: 1.538em;" href="https://help.github.com/articles/github-terms-of-service">GitHub Terms of Service</a><span style="line-height: 1.538em;">.)</span></p>
<p class="p1">However, even with all of the <a href="https://help.github.com/categories/20/articles">resources</a> McCullough’s team has provided, GitHub Pages won’t hold your hand. Participants will need to have GitHub user accounts and a moderate understanding of HTML.&nbsp;</p>
<p class="p3"><strong style="font-size: 1.385em; line-height: 1.538em;">Setting Up Your First Page</strong></p>
<p class="p1">If you have a GitHub account, your first Page already exists—it just doesn’t have anything on it yet. That’s why you need to choose your GitHub username carefully. When you sign up for GitHub, automatically <em>yourname.github.io</em> is reserved for you.&nbsp;</p>
<p class="p1">That’s your top page. But you can have as many GitHub Pages as you want. Just create other repositories. For example, <em>yourname.github.io/blog</em>. GitHub differentiates this by calling the former a User Page, and the latter a Project Page.&nbsp;</p>
<p class="p1">“You can have one repository that will publish at that top level page,” said McCullough. “It makes a great home page, resume, bio, or primary site. Then, you can build sub-sites that center on interests or a blog.”</p>
<p class="p1">In order to create this first, main page, you’ll need to create a new repository. Go to your GitHub user page, tab over to Repositories, and click the green “New” button.&nbsp;</p>
<p class="p1">The name of your new repository must be <em>yourname.github.io</em>, or else the Pages generator won't work. So for me, that's <em>laurenorsini.github.io</em>. Later, if I want to change it to something else like www.lorsini.com, I can redirect a domain name. But this is still where the site will live.</p>
<div class="layout-object large">
<p class="p1"><span class="embedded-Media-image img-caption-c ">
	
			<img src="http://readwrite.com/files/create_repo.png" style="" alt="" width="1000" height="562">
	
	
	</span>
</p>
</div>
<p class="p1">On the new repository’s page, click on the Settings icon in the lower right-hand corner.</p>
<div class="layout-object large">
<p class="p1"><span class="embedded-Media-image img-caption-c ">
	
			<img src="http://readwrite.com/files/repo_settings.png" style="" alt="" width="1000" height="562">
	
	
	</span>
</p>
</div>
<p class="p1">Scroll down until you see GitHub Pages, right under Features. Click “Automatic Page Generator.”</p><div class="breakout lyra-ad say-verso-stripe" id="lyra_lite"></div>
<div class="layout-object large">
<p class="p1"><span class="embedded-Media-image img-caption-c ">
	
			<img src="http://readwrite.com/files/page_generator.png" style="" alt="" width="1000" height="563">
	
	
	</span>
</p>
</div>
<p class="p1">GitHub will automatically populate your homepage with a lot of helpful advice about using Pages. You can leave this for now. Click the “Continue to Layouts” button at the bottom of the page.</p>
<p class="p1"><span class="embedded-Media-image img-caption-c ">
	
			<img src="http://readwrite.com/files/create_page.png" style="" alt="" width="1000" height="563">
	
	
	</span>
</p>
<p class="p1">There are only a few to choose from so far. Pick the one you like best. Keep in mind that it’ll be difficult to change it later. If you make changes to one theme and then decide on another, you’ll lose those changes. Luckily, this is GitHub. Every time you commit a change, there’s the option to revert back to any commit in the page’s history.</p>
<div class="layout-object large">
<p class="p1"><span class="embedded-Media-image img-caption-c ">
	
			<img src="http://readwrite.com/files/themes.png" style="" alt="" width="1000" height="563">
	
	
	</span>
</p>
</div>
<p class="p1">Navigate to <em>yourname.github.io</em>. It may take up to ten minutes, but your page will be live.&nbsp;</p>
<h2 class="p4"><strong>Editing Your First Page</strong></h2>
<p class="p1">In order to update the page’s content, go back to the repository (you can also click the “View on GitHub” button on your page) and navigate to Index.html.&nbsp;</p>
<div class="layout-object large">
<p class="p1"><span class="embedded-Media-image img-caption-c ">
	
			<img src="http://readwrite.com/files/index.png" style="" alt="" width="1000" height="562">
	
	
	</span>
</p>
<p class="p1">Now, it’s time to edit the raw code. I recommend GitHub’s built-in Zen mode, which causes the editor to take up your entire screen. You can also replace everything here with code you've written previously on your computer.&nbsp;</p>
<p class="p1"><span class="embedded-Media-image img-caption-c ">
	
			<img src="http://readwrite.com/files/zen.png" style="" alt="" width="1000" height="563">
	
	
	</span>
</p>
</div>
<p class="p1">When you’re done, finalize your changes with the “Commit Changes” button. In up to ten minutes, you’ll see those changes live.&nbsp;</p>
<div class="layout-object large">
<p class="p1"><span class="embedded-Media-image img-caption-c ">
	
			<img src="http://readwrite.com/files/commit.png" style="" alt="" width="1000" height="563">
	
	
	</span>
</p>
</div>
<p class="p1">If you want to do anything more advanced, like add images or CSS, you’ll need to make a copy of your page on your computer. You’ll also need to have <a href="http://mac.github.com/">GitHub for Mac</a> or <a href="http://windows.github.com/">GitHub for Windows</a>.&nbsp;</p>
<p class="p1">In the lower right-hand corner of the repository page, click “Clone in Desktop.” This will save a copy of your repository in the top-level folder on your computer. It will also open up your copy of GitHub for Mac or GitHub for Windows.</p>
<div class="layout-object large">
<p class="p1"><span class="embedded-Media-image img-caption-c ">
	
			<img src="http://readwrite.com/files/clone.png" style="" alt="" width="1000" height="562">
	
	
	</span>
</p>
</div>
<p class="p1">Find the repository folder by searching your computer, or by looking in the top-level folder. It will, of course, be named <em>yourname.github.io</em>.&nbsp;</p>
<p class="p1">I want a headshot on my GitHub Pages home page, so I dragged and dropped a photo I already had into the folder.&nbsp;</p>
<div class="layout-object large">
<p class="p1"><span class="embedded-Media-image img-caption-c ">
	
			<img src="http://readwrite.com/files/folder.png" style="" alt="" width="1000" height="563">
	
	
	</span>
</p>
</div>
<p class="p1">Then I committed the change on GitHub for Mac. I pressed “Sync Commits” so this change would be reflected on GitHub as well as on my computer. You'll see in the photo, it says "No unsynced commits" once everything is online.</p>
<div class="layout-object large">
<p class="p1"><span class="embedded-Media-image img-caption-c ">
	
			<img src="http://readwrite.com/files/github_mac.png" style="" alt="" width="1000" height="563">
	
	
	</span>
</p>
</div>
<p class="p1">Finally, I edited the code of Index.html to add a photo with HTML:</p>
<pre class="p1">&lt;img align="left" src="identity_square.jpg" width="250" height="250" /&gt;</pre>
<p class="p1">I can see the purists cringing since I really should have adjusted the height, width, and alignment in the CSS to reduce bloat, but I’m just trying to show you the easiest possible process.&nbsp;</p>
<p class="p1">You can view the changes live on <a href="http://laurenorsini.github.io/">my page</a>.&nbsp;</p>
<p class="p2">Of course, this page is far from done. Are you a Pages user? I’d love to see your page linked in the comments.&nbsp;</p>
